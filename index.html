The (operative) difference between a slice and array

Posted on August 7, 2015

When I started with Go, I wondered where all the arrays went - all the examples used something called a slice. But Go certainly has data structures called arrays as well. Behind-the-scenes, both seem to be trivial variants of the traditional CS array data structure. So what’s the difference? Let’s talk declaration first - A slice is declared as follows: var av = []int{1,5,2,3,7} //# slice var bv = [5]int{1,5,2,3,7} //# array : has length in it's type var av = [...]int{1,5,2,3,7} //# array : has length in it's type, //#compiler translates this into "var bv = [5]int{1,5,2,3,7}" So what’s the difference between a slice and an array at runtime?
